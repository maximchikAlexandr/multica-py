# Contract: Quality Baseline Tooling

## Purpose

This contract defines the only allowed format and command interfaces for the immutable pre-refactor test baseline.

## `scripts/check_coverage.py`

CLI:

```text
python scripts/check_coverage.py --coverage-json <path>
```

Behavior:

1. Read `[tool.coverage.regexs]` and `[tool.coverage.thresholds]` from `pyproject.toml`.
2. Read coverage.py JSON from `--coverage-json`.
3. Match files by the configured regexes.
4. Calculate statement percentage for each zone using covered/missing statement counts from the JSON.
5. Fail with exit 1 when a zone is absent or below its threshold.
6. Print one deterministic line per zone in sorted zone-name order.
7. Do not read `tests/_coverage_guard.py` or `tests/_manifest_coverage.py`.

## `scripts/capture_test_baseline.py`

CLI:

```text
python scripts/capture_test_baseline.py \
  --source-sha <40-lowercase-hex> \
  --coverage-json <path> \
  --junit-xml <path> \
  --output tests/quality-baseline.json
```

Behavior:

1. Require `--source-sha` to resolve to the exact pre-change `HEAD` captured before any feature file was created.
2. Require that no production, test, workflow, dependency or marker file differs from `--source-sha`; the only permitted differences are the three baseline scripts, `contracts/multica-live-target.toml`, and the output file.
3. Store `--source-sha` verbatim as `git_sha`; it identifies the source tree measured by the baseline and is never rewritten to a later PR commit.
4. Run `pytest --collect-only -q` separately for `tests/unit`, `tests/contract`, `tests/integration`, `tests/packaging`, and `tests/live`; count collected items by layer.
5. Record `tests/integration` under baseline key `component_source`; comparison maps it only to final `tests/component`.
6. Calculate `mandatory_offline` as all collected items from unit+contract+integration+packaging minus exactly these five excluded low-signal node IDs:
   - `tests/integration/test_issue_workflows.py::test_client_construction`;
   - `tests/integration/test_issue_workflows.py::test_client_context_manager`;
   - `tests/integration/test_streaming_commands.py::test_managed_process_argv`;
   - `tests/integration/test_streaming_commands.py::test_managed_process_stdout_lines`;
   - `tests/integration/test_streaming_commands.py::test_managed_process_consumes_partial_output`.
7. Fail capture if any excluded node ID is absent, duplicated, or not collected exactly once.
8. Count Python logical lines with one fixed rule: non-empty, non-comment physical lines.
9. `tests_python` covers `tests/**/*.py`; `scripts_python` covers `scripts/*.py`; `resource_test_support` covers only `tests/integration/resources/**/*.py` at capture time.
10. Parse total offline duration and test count from the supplied JUnit XML.
11. Copy total and zonal statement/branch values from supplied coverage JSON.
12. Record package install path count as integer `12` for the pre-refactor baseline.
13. Serialize UTF-8 JSON with sorted keys, indentation 2, LF newline and no timestamps.
14. If the output file is absent, write it. If it exists, generate using the supplied `--source-sha` and exit 0 only when generated bytes are identical; otherwise exit 1 without modifying the file.
15. No overwrite or SHA-replacement flag exists.

Schema:

```json
{
  "schema": 1,
  "git_sha": "<40 lowercase hex>",
  "collected": {
    "unit": 0,
    "contract": 0,
    "component_source": 0,
    "packaging": 0,
    "live": 0
  },
  "mandatory_offline": 0,
  "loc": {
    "tests_python": 0,
    "scripts_python": 0,
    "resource_test_support": 0
  },
  "offline": {
    "tests": 0,
    "duration_seconds": 0.0
  },
  "coverage": {
    "statement_percent": 0.0,
    "branch_percent": 0.0,
    "zones": {
      "errors": 0.0,
      "models": 0.0,
      "resources": 0.0,
      "transport": 0.0
    }
  },
  "package_install_paths": 12
}
```

## `scripts/check_test_baseline.py`

CLI:

```text
python scripts/check_test_baseline.py --baseline <path> --mode self-check
python scripts/check_test_baseline.py --baseline <path> --mode compare --stage PR-02
python scripts/check_test_baseline.py --baseline <path> --mode compare --stage PR-03
python scripts/check_test_baseline.py --baseline <path> --mode compare --stage PR-07
python scripts/check_test_baseline.py --baseline <path> --mode compare --stage final
```

`self-check` validates schema, full SHA, non-negative values, deterministic serialization, and that baseline `git_sha` is an ancestor of current `HEAD`.

`compare` always verifies that baseline file bytes are unchanged and that current mandatory offline count is not below baseline `mandatory_offline`. Stage-specific checks are fixed:

- `PR-02`: mandatory offline count only;
- `PR-03`: mandatory offline count plus current `tests/component/resources/**/*.py` logical LOC at most 75% of baseline `resource_test_support`;
- `PR-07`: all `PR-03` checks plus package install paths exactly 6;
- `final`: all `PR-07` checks plus zonal coverage at or above configured thresholds.

Unknown or omitted `--stage` with `--mode compare` exits 2.

No command updates the committed baseline after `PR-01`.
